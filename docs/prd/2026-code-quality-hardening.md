# 코드 품질 강화 (Review & Hardening) — 2026

## Problem Statement

행사 수집·정규화·매크로·알람·추천 파이프라인은 실동작이 검증된 상태(단위 테스트 통과, 오프라인 엔드투엔드 완주)지만, 운영 신뢰성을 좌우하는 비기능 품질에 공백이 있다. 진단 출력이 구조화되지 않아 자동화(CI/cron) 환경에서 실패 원인을 grep으로 추적해야 하고, 외부 공공 API·LLM 호출이 일시적 장애(타임아웃·5xx)에 재시도 없이 즉시 폴백으로 강등되어 데이터가 조용히 누락될 수 있으며, 행사 레코드가 구조 없는 사전(dict)으로 흐르면서 키 오타가 런타임까지 잠복한다. 또한 신청 매크로의 핵심 안전 불변(비자동 작업의 자동 제출 차단)이 브라우저 미설치 환경에서는 검증되지 않아, 무인 실행 중 의도치 않은 신청 제출 위험이 회귀해도 잡아내지 못한다.

이 작업은 새 기능이 아니라, 무료·무인 운영 인프라가 신뢰할 수 있게 축적·개선되도록 기존 코드의 관측성·견고성·안전성을 끌어올린다.

## Solution

근거(실행 로그·재현 가능한 검증)에 기반한 우선순위로 품질을 단계적으로 강화한다. 진단 출력을 레벨 분리된 구조화 로그로 옮겨 실패를 빠르게 진단하고, 외부 호출에 공통 재시도·백오프 정책을 적용해 일시 장애에 견디게 하며, 행사 레코드와 추천 점수에 명시적 타입 계약을 부여해 오류를 조기에 드러낸다. 매크로의 안전 불변은 가짜(fake) 브라우저 대역을 통해 브라우저 없이도 상시 검증되도록 만든다. 사용자에게 노출되는 출력 계약(알람 발송 표준출력 등)은 회귀 위험을 피하기 위해 보존한다.

## User Stories

1. 운영자로서, 무인 cron 실행이 실패하면 로그 레벨(INFO/WARNING/ERROR)만으로 어느 단계에서 무엇이 폴백·스킵·실패했는지 즉시 식별하고 싶다. 그래야 grep 추적 없이 빠르게 복구한다.
2. 운영자로서, 공공 API나 LLM이 일시적으로 타임아웃·서버 오류를 내도 자동 재시도로 데이터를 끝까지 수집하고 싶다. 그래야 일시 장애로 행사가 조용히 누락되지 않는다.
3. 개발자로서, 행사 레코드의 속성에 명시적 타입 계약이 있길 원한다. 그래야 키 오타나 형태 불일치를 편집기·정적 검사에서 잡고 런타임 잠복을 막는다.
4. 추천 로직 담당자로서, 점수 가중치가 흩어진 매직넘버가 아니라 한곳에 정의된 상수이길 원한다. 그래야 가중치 튜닝과 A/B 실험을 단일 지점에서 안전하게 한다.
5. 안전 책임자로서, 신청 매크로가 자동 제출이 허용되지 않은 작업에서 절대 폼을 제출하지 않음을 브라우저 설치 여부와 무관하게 CI에서 보장받고 싶다. 그래야 무인 실행 중 오제출 회귀를 출시 전에 차단한다.
6. 기여자로서, 코드가 일관된 스타일 규약(한 줄 복합문 없음 등)을 따르길 원한다. 그래야 리뷰가 동작 변경에 집중되고 디프 잡음이 줄어든다.
7. 메인테이너로서, 알람 발송의 표준출력 계약처럼 테스트로 고정된 사용자 표면은 리팩터링 중에도 깨지지 않길 원한다. 그래야 관측성 개선이 기존 동작을 회귀시키지 않는다.

## Implementation Decisions

### Modules

- **공통 HTTP 헬퍼** — 외부 JSON 호출을 위한 단일 진입 함수. 재시도 대상(타임아웃·5xx)과 비재시도(4xx 클라이언트 오류)를 구분하고, 재시도 횟수·백오프·타임아웃을 파라미터로 노출하는 깊고 단순한 모듈. AI 추천기와 각 원격 수집 어댑터가 공유한다. (구현·검증 완료)
- **구조화 로깅** — 모듈별 네임드 로거. 진행=INFO, 폴백/스킵=WARNING, 매핑·발송 실패=ERROR. 엔트리포인트에서 한 번만 기본 설정을 수행한다. (파이프라인 오케스트레이터·수집 베이스 적용 완료, 잔여 확대는 제품 결정 후)
- **행사·점수 타입 계약** — 행사 레코드를 위한 구조화 타입 정의와, 추천 점수+근거 묶음을 위한 데이터 구조. 사전을 흘려보내던 시그니처를 교체해 정적 검사 대상으로 만든다. (보류 — 정적 타입 검사 CI 도입과 묶어 별도 슬라이스)
- **추천 가중치 상수** — 지역·테마 등 점수 가중치를 모듈 상수로 추출해 추천 튜닝의 단일 지점으로 만든다. (완료)
- **매크로 실행기 안전 분기** — 단계 필터링과 구동 로직을 추출해, 가짜 페이지 대역(스파이)으로 "비자동 작업 제출 제거 / 일시정지 단계에서 정지" 불변을 브라우저 없이 검증한다. 실브라우저 경로는 선택 마커로 유지. (완료)

### Technical Choices

- 외부 호출 재시도는 멱등 읽기 성격의 수집·생성 호출에 한정하고, 클라이언트 오류(4xx)는 재시도하지 않아 잘못된 요청을 증폭시키지 않는다. 모든 외부 의존은 무키·오프라인 상황에서 규칙기반/픽스처 폴백으로 강등되는 기존 설계를 유지한다.
- 로깅 전환은 진단 출력에만 적용한다. 알람 발송의 DRY/SENT 표준출력처럼 테스트로 고정된 사용자 계약은 의도적으로 보존하며, 이를 구조화 로그로 옮길지는 별도 제품 결정 사항이다.
- 타입 계약은 점진적으로 도입하되, 정적 타입 검사기를 CI에 추가하는 작업과 함께 한 슬라이스로 묶어 검증 가치를 확보한 뒤 적용한다.
- 매크로 안전 검증은 브라우저 자동화를 추상 인터페이스 뒤에 두어 결정론적 단위 테스트로 핵심 안전 불변을 상시 보장하고, 실브라우저 통합은 옵션으로 분리한다.

## Testing Decisions

- 무인·무료 인프라 특성상 외부 의존 없이 결정론적으로 도는 단위 테스트를 1급으로 둔다. 네트워크·브라우저·키가 필요한 경로는 가짜 대역과 폴백으로 검증하고, 실연결 테스트는 선택 마커로 격리한다.
- 커버리지 필수 모듈: 공통 HTTP 헬퍼(재시도/비재시도 분기, 백오프), 매크로 실행기 안전 불변(제출 차단·일시정지 정지), 추천 점수 가중치. 신규 테스트로 이미 확장됨(전체 70 통과 / 3 스킵, 직전 58 대비 +12).
- 회귀 가드: 알람 표준출력 계약 등 사용자 표면을 고정한 기존 테스트는 리팩터링 후에도 그대로 통과해야 한다. 오프라인 엔드투엔드 완주(행사 10건 → 인덱스 → SQLite → wiki)를 통합 스모크로 유지한다.
- 타입 계약 슬라이스 진행 시 정적 타입 검사를 CI 게이트로 추가해 키 오타·형태 불일치를 빌드 단계에서 차단한다.

## Out of Scope

- 신규 사용자 기능(검색 축·지도·알람 채널·AI 플랜 형식 등) 추가나 변경.
- 알람 발송 표준출력 계약을 구조화 로그로 이전(별도 제품 결정 전까지 보존).
- 결제·티켓팅·SSO 등 1차 비범위 항목.
- 유료 API·외부 DBMS 도입(무료·flat-file 원칙 유지).
- 성능 최적화·대규모 리아키텍처(본 작업은 비기능 품질 강화에 한정).

## Further Notes

- 완료 상태(검증된 근거 포함): PEP8 복합문 정리 + 가중치 상수화, 공통 HTTP 재시도/백오프 + 신규 테스트, 매크로 안전 불변 CI 검증, 핵심 경로 로깅 전환.
- 잔여/의존: 타입 계약(구조화 타입 + 점수 데이터 구조) 도입은 정적 타입 검사 CI와 묶어 다음 슬라이스로. 로깅 전면 확대는 표준출력 계약 이전 여부에 대한 제품 결정에 의존.
- 검증 명령 기준선: 단위 테스트 70 통과/3 스킵, 오프라인 파이프라인 엔드투엔드 완주, 무키 AI 폴백 정상.
