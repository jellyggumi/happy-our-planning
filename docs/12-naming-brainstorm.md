# 네이밍 & 소개글 브레인스토밍

기존 코드네임 `Happy Our Planning`(repo: `happy-our-planning`)을 유지하되,
사용자에게 노출되는 **제품 브랜드명**을 페르소나·컨셉·주요기능에 맞춰 정의한다.

## 1. 페르소나 (컨셉에서 도출)

**"좋은 기회를 자꾸 놓치고 뒤늦게 후회하는 사람."**
공연·축제·전시·교육·공모·정부지원 같은 혜택이 흩어져 있어, 신청 마감을 넘기거나
존재 자체를 몰라서 놓친다. 대표 인물: 바쁜 직장인, 육아 부모, 취준생, 지역
문화·지원 혜택을 챙기고 싶은 주민.

## 2. 컨셉 한 줄

전국 행사를 **지도에서 발견 → 매크로로 신청 → 알람으로 마감 방어 → AI가 주간
일정으로 큐레이션**하는, "기회를 대신 챙겨주는 개인 비서". 무료·경량 인프라.

## 3. 주요기능 (네이밍 앵커)

5축 검색(지역·기간·나이·테마·키워드) · 신청 매크로 · 마감/신규/결과 알람 ·
AI 추천 플래닝.

## 4. 세 관점 아이디어 (각 5개)

**PM (시장적합·차별화)**
1. 놓치마 (Notchima) — "놓치지 마". 마감/기회 놓침 방지라는 핵심가치 직결
2. 다챙겨 — "다 챙겨준다" 비서 컨셉
3. 기회알리미 — 알람 가치 전면
4. 혜택지도 — 정부지원/혜택 + 지도 발견
5. 온행사 (On행사) — "모든 행사 ON"

**Designer (기억성·발음·아이콘)**
1. 노치 (Notch) — 놓치마 축약, 앱 아이콘 친화
2. 골라줌 — 큐레이션 어감
3. 행사픽 (HaengsaPick) — 선택/추천 직관
4. 마이행사 — 개인화 강조
5. 콕행사 — "콕 집어주는" 추천

**Engineer (도메인·핸들·국제화)**
1. eventory (event + inventory) — flat-file 지식 DB 어감
2. nochima(.app) — 브랜드 일치 핸들
3. haengsa — 순수 한글 로마자, 충돌 적음
4. eventmap-kr — SEO 직관
5. okf-events — 데이터 표준 강조(내부용)

## 5. 최종 선정: **놓치마 (Notchima)**

- **핵심가치 직결**: 제품의 차별점은 "신청 매크로 + 마감 알람 + AI 플랜"으로
  *기회를 놓치지 않게* 하는 것 → 이름이 곧 가치 제안.
- **기억성·발음**: 한글 3음절, 즉시 의미 전달. 로마자 핸들 `notchima` 확보 쉬움.
- **확장성**: "노치"로 축약 가능(앱/아이콘).

**검증할 가정**: (a) 타깃이 "놓치마"를 명령형 비서로 긍정 인지하는가,
(b) 검색/혜택 탐색 맥락에서 상기되는가, (c) 로마자 핸들·도메인 가용성.
